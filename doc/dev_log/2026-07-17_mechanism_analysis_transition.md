# Motivation mechanism-analysis transition — 2026-07-17

The user accepted the frozen recurrence-dominant observation as the end of the
first-round Motivation experiment and authorized the repository to enter
mechanism analysis.

The active question is no longer whether the four Qwen variants establish
strict transfer; the frozen result already says they do not at the registered
single seed and retrospective KuaiSearch holdout. The new question is which
combination of signal availability, query-aware history selection, preference
abstraction, candidate readout, objective shortcut, or population instability
best explains that observation.

The completed first-round execution prompt and planning document were removed.
`experiments/motivation/protocol.yaml`, result reports, configs, checkpoints,
score bundles, and release hashes remain frozen evidence. Current work is
governed by `experiments/motivation/mechanism_analysis_plan.md` and stops after
a bounded mechanism evidence matrix; it does not yet authorize a solution
architecture.

The user further set the research priority: data/supervision/power analysis
must remain architecture-facing rather than become an isolated data study.
Every data-construction result should identify the architectural limitation it
exposes and how the training signal should interact with history routing,
cross-item preference abstraction, candidate readout, or objective shortcuts.
No fixed effort percentage is imposed. Train-only data construction may be
co-designed with a future architecture, but the shared evaluation population
and evidence boundary remain fixed.
