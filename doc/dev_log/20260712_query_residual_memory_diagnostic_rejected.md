# Query-residual source-memory diagnostic rejected

After C46 terminated, one fixed read-only diagnostic used only its already-open
source and A labels. Each early click was represented as the normalized item
component orthogonal to its contemporaneous query; current candidates were
projected relative to the current query by the same formula. No alternative
formula, fallback, or subset was screened.

Only 271/600 target users had an early-source memory. Overall NDCG@10 was
`0.283935` for true memory, `0.289873` for wrong memory, and `0.301470` for the
C46 semantic mean control. True-minus-wrong was `-0.005938` with a zero-crossing
interval, and clicked direction was slightly negative. The historical-query
residual does not remove C46's semantic shortcut in a useful direction.

Decision: do not create a query-residual candidate or tune its projection,
memory window, fallback, or coverage slice. Architecture search returns to the
established exact-recurrence surface rather than forcing another unsupported
non-repeat transfer representation.
