# C67 cross-validated fast weights: data-free terminal

C67 followed the C66 closure by moving personalization into an inner learning
state rather than another output residual. Literature review rejected plain
TTT, TTT4Rec, GradMem, HyperLoRA, and gradient agreement as novelty. The locked
primitive admitted an event update only when its exact one-step learner
improved reconstruction of other history events, with the closest reductions
trained at identical capacity.

The mechanics worked, but the scientific object did not. All models solved the
synthetic ranking task at roughly 96% while preserving the same accuracy under
wrong history. The primary assigned useful and nuisance events nearly uniform
mass, did not abstain when every event came from a different law, and did not
beat ordinary TTT or its first-order reduction. This is a generic
query-candidate shortcut, caught before any repository record or label.

This tightens the next architecture boundary: excluding query/candidate from
the write is insufficient if a flexible read encoder can turn almost any
history-written state into the same generic comparator. A future primitive
must establish functional dependence on the identity of the written state by
construction and bind a history-scrubbed function-equivalence control before
training. Simply removing one C67 edge or changing its synthetic generator
would be an outcome-informed rescue and is forbidden.
