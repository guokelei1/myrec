# 2026-07-12 — C68 preimplementation direction

C67 closed before repository data because a history-only fast-weight writer
still supported a generic query/candidate read.  The post-terminal audit then
rejected empirical-Fisher and inverse-Fisher drafts: the former is a quadratic
kernel over event gradients and the latter is ridge/KRR in gradient-feature
space.  Pooled global-minus-user residuals and semantic/collaborative agreement
were also rejected as direct neighbors rather than registered as innovations.

The remaining C68 hypothesis follows the C42/C43 cross-domain boundary.  C42
showed useful, user-specific query/history transport on Amazon; C43 showed that
the same operator behaves mainly as generic history expansion on KuaiSearch.
C68 therefore compares a user's conditional event free energy with an
exchangeable population-event free energy, then subtracts the candidate-free
version of the same contrast.  Query/candidate-only and separable
query-history terms cancel exactly.

Only a data-free G0 is authorized.  It binds finite-temperature free energy to
mean, single-NULL, user-only, and pooled reductions, plus wrong-history,
unsupported, equal-set, fixed-carrier, reference-only, no-history, repeat,
determinism, and candidate-permutation attacks.  A pass would authorize an
implementation review, not repository data.
