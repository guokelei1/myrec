Each candidate writes only to its own `systems/<candidate>/` tree and to
run/model/artifact locations bearing its assigned prefix. It must not reuse
another candidate's writable output directory or mutate a predecessor's
frozen files. A component may move to `src/myrec/` only after a review confirms
that promotion cannot couple hypotheses or leak outcomes.

## Candidate-local layout

Each track owns its structure from the first source line:

```text
systems/<candidate>/
  configs/        exploratory and preregistered candidate configs
  model/          Transformer/LM ranking core and candidate mechanism
  train/          candidate-local training/probe entry points
  tests/          candidate-local tests and tiny fixtures
  notes/          design, nearest-neighbor table, and frozen gate
```

No early proposed-system prototype belongs in `src/myrec/models/`, and no
exploratory candidate config belongs in top-level `configs/`. A winning,
reviewed candidate may later promote reusable code and frozen production
configs to those shared locations.

## C75--C80 terminal sequence

C75 froze the pretrained semantic carrier and trained only the two-hop routing
maps.  Its outputs were highly rank- and wrong-history-active, but the primary,
pooled, and factual modes improved their fixed training anchor loss by only
about `0.01%--0.02%`, far below the frozen `0.5%` requirement.  It closed
before validation labels; freezing semantic coordinates removed drift but did
not yield a learnable ranking operator.

The subsequent source-observability audit separated representation failure
from source absence.  Kuai pooled full/text/ID histories were negative and
Amazon pooled text reached only `true-null +0.001661`; ordinary Amazon
full-token BGE reached `+0.025298` with positive CI and `true-wrong +0.035944`.
Weight-frozen edge attribution showed that bidirectional C--H, Q--H, and
history-read-context paths were load-bearing, while event order was not.

C76's counterfactual layer trajectory failed its data-free gate because a
candidate-only shortcut appeared in the factual/cut difference and every
control failed the held-out nuisance reversal.  C77's frozen query-authenticated
subgraph rejected that shortcut and passed clean/wrong/query-mask safety, but
shuffle retained only `35%--39%` of clean margin and a simpler query filter
tied it.  C78 removed event-index positions and restored exact set behavior;
its primary was strong, but the `triadic_set` control matched or beat it in all
seeds.  These three candidates closed before repository labels.

C80 promoted the C78 winning control to the terminal pretrained-LM real gate.
All 15 fixed fits were finite, decreased loss, preserved frozen anchors, had
active backbone/head gradients, exact candidate permutation, and exact
no-history base behavior.  Its defining event-permutation error was
`0.031869/0.068417/0.039372` against `2e-6`; a label-free fp32 diagnostic still
measured `3.34e-6`.  C80 therefore closed before opening its 365 fresh labels.
The authority is `reports/pps_c80_amazon_real_gate.json`; utility is unknown,
not negative.

## Git policy

Track source code, configs, and design notes. Do not track checkpoints,
runs, logs, or caches - the `.gitignore` catches:

```text
systems/**/checkpoints/
systems/**/runs/
systems/**/logs/
systems/**/outputs/
systems/**/.cache/
```

The source directories are candidate-local.  Registration here does not
authorize GPU use, train-internal reruns, dev evaluation, full training, or
test access.
