# C63 pre-implementation review

Decision: **authorize only implementation, unit tests, and the locked
data-free G0**.

- The successor changes C62's load-bearing binding operator, not slot count,
  width, optimizer, or training budget.
- Event-wise conservation is an exact structural invariant and has a clear
  representation intervention: duplicate write mass is impossible.
- The design remains a Transformer/LLM ranker; the finite allocator writes
  internal memory and does not route fixed scores.
- The mechanism is close to Slot Attention, MESH, stick-breaking attention,
  and Ordered Memory.  All are acknowledged; three executable binding controls
  make novelty conditional on empirical rent.
- A four-interest plus nuisance G0 is necessary because a two-interest clean
  task would not make the NULL remainder or full slot capacity load-bearing.
- No repository data, fit label, fresh role, dev, test, or qrel is authorized
  before G0 passes all three seeds.

Passing G0 would establish only a viable binding primitive.  Real utility and
cross-domain value would still require a new execution lock and common
dual-domain gate.
