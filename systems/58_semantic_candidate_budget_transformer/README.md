# C58 — Semantic candidate-budget Transformer

C58 is the fixed-direction formulation test authorized by C57's terminal
result.  It retains candidate-axis history allocation but removes the
unidentified trainable value/output gauge: frozen contextual LM token
similarities determine both evidence assignment and ranking direction.

This is not a claim that ColBERT, Slot Attention, semantic matching, or
request standardization is new.  It is a bounded test of whether their
constrained PPS composition pays rent over no-NULL Slot, ordinary
history-axis, pooled, raw-query, and wrong-history controls.

The first stage is label-free and reuses the untouched C56 holdout.  Only a
passed mechanics gate may open its compact fit labels.  C26 A/B/escrow,
dev/test, qrels, and every fresh role remain closed.
