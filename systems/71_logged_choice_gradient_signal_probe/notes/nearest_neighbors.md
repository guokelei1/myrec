# C71 nearest-neighbor boundary

C71 claims no novelty. It is an information test for C70.

| Neighbor | Relation to C71 |
|---|---|
| XDM / impression-aware recommendation | actual shown-but-unclicked items are established useful inputs; C71 cannot claim impressions as new |
| multinomial-logit gradient | selected state minus slate expectation is the standard score-gradient direction |
| DeltaNet / fast-weight memory | C70's possible rank-one write is a direct associative-memory neighbor |
| RTM | established joint Transformer matching over query, user evidence, and item evidence |
| C67 | local history-only fast weights failed functional identity without a signed choice object |
| C69 | semantic-matched item relation is the binding pseudo-negative information control |

Primary references:

- XDM: https://arxiv.org/abs/2010.12837
- Impression-Aware Recommender Systems: https://arxiv.org/abs/2308.07857
- DeltaNet: https://arxiv.org/abs/2406.06484
- RTM: https://arxiv.org/abs/2004.09424

A C71 pass would justify data acquisition and a separate C70 architecture
review. It would not establish that the gradient formula or fast weights are a
paper contribution.
