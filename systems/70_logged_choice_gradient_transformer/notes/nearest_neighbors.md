# C70 nearest-neighbor boundary

C70 makes no novelty claim at the data gate.

| Neighbor | Overlap | Binding distinction/control |
|---|---|---|
| XDM | learns from impressed-but-unclicked sequences | direct impression-aware control; C70's possible increment is a query-conditioned signed function write, not the use of unclicked items |
| impression-aware recommenders | broad family using shown/non-interacted items | establishes that logged impressions are not novel |
| DeltaNet / fast-weight memory | rank-one associative parameter writes | direct operator neighbor; positive-only and pooled reductions are mandatory |
| RTM | joint Transformer matching of query, user history, and item evidence | strong fine-grained personalized-search neighbor |
| C67 | history-only fast weights with held-out self-validation | local negative result; C70 changes the write target to an observed within-slate choice gradient |
| C69 | anchored item relation with semantic-matched negatives | mandatory information-object control |

Primary sources:

- XDM: https://arxiv.org/abs/2010.12837
- Impression-Aware Recommender Systems: https://arxiv.org/abs/2308.07857
- DeltaNet: https://arxiv.org/abs/2406.06484
- RTM: https://arxiv.org/abs/2004.09424

Even a positive signal gate would establish only that logged choice gradients
are useful. A later review must show architectural rent over impression-aware
sequence models and generic fast-weight Transformers.
