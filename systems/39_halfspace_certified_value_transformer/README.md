# C39 Halfspace-Certified Value Transformer

C39 is the first representation/value-interface candidate after the C38
cross-domain falsifier. C38 established that strict-past, query-attended true
history has large transferable ranking value, but its tangent projection was
significantly worse than an equal-capacity unprojected write. C39 therefore
keeps a standard unprojected query-to-history value write as a common backbone
and tests one new primitive inside the Transformer `V/W_O` path: an admitted
candidate/event value may enter a candidate-local query token only after a
closed-form projection removes its component that locally lowers that
candidate's readout score.

C39 is not an Amazon-specific architecture. Dataset, category, query type, and
target-label attributes never select a code path. Missing history or query
gives an exact zero correction, and any exact candidate recurrence suppresses
cross-item correction in favor of the common item-memory base.

The current authorization is design formulation plus a synthetic/operator
falsifier. Real fit labels, train-internal A scores, delayed roles, dev, and
test remain closed until the design gate, proposal lock, G0, and execution lock
pass in that order.
