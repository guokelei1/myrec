# Curated reports

This directory is intentionally empty of historical results while the new
direction is being prepared. New tracked reports must be small, outcome-
oriented, and tied to a reviewed E0–E8 artifact from doc 34.

Keep raw logs, score dumps, checkpoints, and generated tables under
`runs/`, `artifacts/`, or `models/`. Register paper-table numbers only after
copying them verbatim from the shared evaluator's `metrics.json`.

Every development evaluator call is appended to `dev_eval_log.jsonl` and
reconciled with the active budget.
